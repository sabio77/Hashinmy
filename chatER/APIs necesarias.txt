Conceptos de APIs necesarias para memoriaBACKEND que no aparecen como API publicada en `APIS de memoriaBACKEND`.

Estado actualizado 2026-07-04 v22:
- No hay conceptos de APIs pendientes para esta iteración.
- Este archivo se mantiene sin contratos nuevos porque las funciones backend requeridas ya aparecen cubiertas en `APIS de memoriaBACKEND/apis v483.txt` o son procesos locales del frontend.
- No se deben volver a listar aquí como APIs necesarias: `ACCESOSdirectosX`, `CONVERSACIONESx`/acciones de chat, `RELACIONESusuarioX`/perfil-contacto-QR/contactos-QR-resolver, `PERMISOSnavegadorX`, `PWAinstalacionX`, `ImagenesCloudflareR2x`/imagenesR2, `MEDIAfirmadaX`, `STREMEx`, `SENALESefimerasX`, `PRESENCIAusuarioX` ni `MENSAJESx`, mientras sigan publicados en memoriaBACKEND.

Regla de mantenimiento obligatoria:
- Este archivo NO define APIs para procesos locales del frontend.
- Los procesos locales, como compresión WebP, generación/lectura visual de QR, scroll de listas, ventanas de permisos del navegador, UI PWA, manejo visual de adjuntos, barra de progreso local o gracia visual de `Escribiendo...`, se documentan en `docs/BLOQUES_LEGO.md` o en el módulo correspondiente.
- Aquí solo se agregan conceptos de backend cuando una función requiere sincronización entre usuarios/dispositivos, auditoría, almacenamiento persistente, verificación, cuentas, relaciones, métricas o recuperación multiusuario y no existe una API asociada en `APIS de memoriaBACKEND`.
- Si una API ya existe en `APIS de memoriaBACKEND`, este archivo no debe duplicar su contrato; como máximo puede dejarse una nota breve de extensión pendiente no publicada.

Cobertura backend ya existente en memoriaBACKEND:
- Accesos directos de chat: cubierto por `ACCESOSdirectosX` y alias `/api/v1/accesos-directos-chat`.
- Acciones de chat como limpiar, ocultar/restaurar o registrar acciones de conversación: cubierto por `CONVERSACIONESx` y alias `/api/v1/acciones-chat`.
- Perfil por QR, resolución de QR de contacto y creación/restauración idempotente de relación: cubierto por `RELACIONESusuarioX`, `/api/v1/perfil-contacto/qr` y `/api/v1/contactos/qr-resolver`.
- Auditoría de permisos del navegador cuando aplique a métricas multi-dispositivo: cubierto por `PERMISOSnavegadorX`.
- Auditoría de instalación PWA cuando aplique a soporte o métricas: cubierto por `PWAinstalacionX`.
- Imágenes de chat en R2 o media firmada: cubierto por `ImagenesCloudflareR2x`/imagenesR2 y `MEDIAfirmadaX`; el backend recibe únicamente el archivo WebP final ya validado por el frontend.
- Mensajería real, señales efímeras, presencia y escritura: cubierto por `MENSAJESx`, `STREMEx`, `SENALESefimerasX` y `PRESENCIAusuarioX`.

Notas de mantenimiento local 2026-07-04 v22:
- No se agrega API nueva para `IMAGENwebpCOMPRESIONx`. La compresión, redimensionado, comparación de candidatos, verificación RIFF/WEBP, control de calidad visual y garantía `<= 200 KB` son procesos locales del navegador antes de cualquier subida.
- No se agrega API nueva para la compuerta visual limitada por `maxDimension`. El cálculo de dimensión mínima aceptable frente a la política efectiva es una validación local del bloque WebP y del respaldo Canvas, no un proceso backend.
- No se agrega API nueva para reutilización segura de WebP preparado. `reusePreparedWebp()` solo revalida localmente el archivo contra la política efectiva antes de `imagenesR2` o `MEDIAfirmadaX`; no sincroniza datos ni requiere backend nuevo.
- No se agrega API nueva para `QRcodigosX`. El bloque genera/lee el QR visual en el navegador; la sincronización real de contactos, identidad y conversaciones usa las APIs ya publicadas de memoriaBACKEND.
- No se agrega API nueva para `PERMISOSx`. El bloque muestra la ventana local de permisos y guía de configuración; la auditoría remota, si se requiere, usa `PERMISOSnavegadorX`.
- No se agrega API nueva para la ventana flotante de instalación PWA. La UI local usa `beforeinstallprompt`/`appinstalled`; la auditoría remota, si se requiere, usa `PWAinstalacionX`.
- No se agrega API nueva para `Escribiendo...`. La app solo muestra la señal si llega escritura real de la otra parte y mantiene una gracia visual local de 2 segundos; los eventos remotos siguen perteneciendo a `STREMEx`/`SENALESefimerasX`.
- No se agrega API nueva para scroll de listas. Es comportamiento visual local y debe mantenerse en CSS/JS de la interfaz.

Criterio para futuras iteraciones:
- Si una función nueva se puede resolver totalmente dentro del navegador o del estado local sin sincronización entre usuarios, no debe entrar en este archivo.
- Si una función nueva necesita persistencia, verificación, relación entre cuentas, auditoría remota o consistencia multi-dispositivo y no aparece en `APIS de memoriaBACKEND`, se debe agregar aquí solo como concepto funcional, no como proceso local ni paso a paso de frontend.
